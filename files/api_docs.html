Utility
GET /health Service health check ▼

Returns the service status and whether PDF parsing support is available.

Response 200
{ "status": "ok", "pdf_support": true }
Parser
POST /parse/text Parse CV from raw text ▼

Send raw CV text as a JSON body and receive structured data back.

Request body
FieldTypeRequiredDescription
textstringrequiredRaw CV text content
Example request
{ "text": "Jane Doe\njane@example.com\n+1 555 123 4567\n\nExperience\n..." }
Response 200
{ "personal_info": { "name": "Jane Doe", "email": "jane@example.com" }, "work_experience": [{ "title": "...", "company": "..." }], "education": [{ "degree": "...", "institution": "..." }], "skills": { "technical": ["..."], "soft": ["..."] }, "certifications": [], "languages": [] }
Error responses
422 — CV text is empty
POST /parse/file Parse CV from uploaded file ▼

Upload a .txt or .pdf file and receive structured CV data. Content-Type must be multipart/form-data.

Form field
FieldTypeRequiredDescription
filefilerequired.txt or .pdf resume file
Example (curl)
curl -X POST http://localhost:8000/parse/file \ -F "file=@resume.pdf"
Response 200

Same structure as POST /parse/text.

Error responses
415 — Unsupported file type 422 — File is empty
Jobs
POST /add-job Generate job embedding ▼

Generate a 3072-dimensional vector embedding for a job posting. Returns the job document enriched with the embedding. Pass the result to POST /index-job to store it in Elasticsearch.

Request body
FieldTypeRequiredDescription
job_idstringrequiredUnique identifier for the job
job_titlestringrequiredJob title
job_descriptionstringrequiredFull job description text
skillsstring[]requiredList of required skills
Example request
{ "job_id": "job_001", "job_title": "Data Engineer", "job_description": "We are looking for a Data Engineer...", "skills": ["Python", "Spark", "AWS"] }
Response 200
{ "job_id": "job_001", "job_title": "Data Engineer", "job_description": "We are looking for a Data Engineer...", "skills": ["Python", "Spark", "AWS"], "embedding": [0.0412, -0.0231, ... 3072 floats total] }
POST /index-job Index job into Elasticsearch ▼

Persist a JobDocument (including its embedding) into Elasticsearch. Call POST /add-job first to generate the embedding, then pass the full response here.

Request body

Full JobDocument — same shape as the POST /add-job response, including the embedding array.

Response 200
{ "message": "Job indexed successfully", "job_id": "job_001" }
PATCH /update-job/{job_id} Partially update a job ▼

Partially update a job document in Elasticsearch. Only fields you provide are changed — omitted fields are left intact.

Path parameter
ParamTypeDescription
job_idstringThe job's unique ID
Request body (all optional)
FieldTypeRequiredDescription
job_titlestringoptionalNew job title
job_descriptionstringoptionalNew job description
skillsstring[]optionalReplacement skills list
Response 200
{ "updated": true, "job_id": "job_001", "result": "updated" }
Error responses
404 — Job not found 422 — No update fields provided
DELETE /delete-job/{job_id} Delete a job ▼

Permanently remove a job document from Elasticsearch by its job_id.

Path parameter
ParamTypeDescription
job_idstringThe job's unique ID
Response 200
{ "deleted": true, "job_id": "job_001", "result": "deleted" }
Error responses
404 — Job not found
Seekers
POST /seeker-embedding Generate seeker embedding ▼

Generate a 3072-dimensional vector embedding for a job seeker from their title and skills. The embedding has the same dimensionality as a job embedding and can be used directly for similarity search.

Request body
FieldTypeRequiredDescription
titlestringrequiredJob seeker's current or desired title
skillsstring[]requiredList of the seeker's skills
Example request
{ "title": "Data Engineer", "skills": ["Python", "Spark", "AWS"] }
Response 200
{ "title": "Data Engineer", "skills": ["Python", "Spark", "AWS"], "embedding": [0.0231, -0.0187, ... 3072 floats total] }
POST /index-seeker Index seeker into Elasticsearch ▼

Persist a SeekerDocument into Elasticsearch. Call POST /seeker-embedding first, then add a seeker_id to the response and pass it here.

Request body
FieldTypeRequiredDescription
seeker_idstringrequiredUnique identifier for the seeker
titlestringrequiredSeeker's title
skillsstring[]requiredSeeker's skills
embeddingfloat[]required3072-dim embedding from /seeker-embedding
Response 200
{ "message": "Job seeker indexed successfully", "seeker_id": "abc123" }

Search
POST /search-query Embed a search query ▼

Accepts a plain-text search query and a seeker's embedding, and returns a joint embedding that blends the search intent with the seeker's profile via element-wise averaging. The result is used as input to POST /similarity.

Request body
FieldTypeRequiredDescription
querystringrequiredFree-text search query
seeker_embeddingfloat[3072]required3072-dim embedding from /seeker-embedding
Response 200
{ "query": "machine learning engineer with Python experience", "embedding": [0.0198, -0.0054, ... 3072 floats total] }
Error responses
422 — query must not be empty 422 — seeker_embedding must have 3072 dimensions
POST /similarity ANN vector search (kNN) ▼

Run an Approximate Nearest Neighbour (kNN) search against the jobs index using a query embedding from POST /search-query. Returns the top-k most similar jobs ranked by cosine similarity score.

Request body
FieldTypeRequiredDescription
query_embeddingfloat[3072]required3072-dim embedding from /search-query
kintegeroptionalNumber of results to return. Default: 10
num_candidatesintegeroptionalANN candidate pool size. Higher = more accurate but slower. Default: 100
Response 200
{ "total": 3, "results": [ { "job_id": "job_001", "job_title": "Data Engineer", "job_description": "...", "skills": ["Python", "Spark"], "score": 0.9421 } ] }
Error responses
422 — query_embedding must have 3072 dimensions 502 — kNN search failed
Skills & ESCO
POST /normalize Normalize title & skills against ESCO ▼

Fuzzy-match a raw job title and skills list against the ESCO dataset and return the closest standardised label with a confidence score. Confidence = 60% title + 40% average skill score.

Request body
FieldTypeRequiredDescription
titlestringrequiredRaw job title to normalize
skillsstring[]requiredList of raw skills to normalize
Example request
{ "title": "Data Engineer", "skills": ["Python", "Spark", "AWS"] }
Response 200
{ "normalized_title": "data engineer", "occupation_uri": "http://data.europa.eu/esco/occupation/...", "confidence": 0.87, "title_score": 92.0, "matched_skills": [ { "input_skill": "Python", "matched_skill": "Python (programming language)", "score": 0.95 } ] }
POST /skill-gap Identify missing skills ▼

Exact case-insensitive comparison between job skills and candidate skills. Returns only the skills the candidate is missing.

Request body
FieldTypeRequiredDescription
job_skillsstring[]requiredSkills required by the job
candidate_skillsstring[]requiredSkills the candidate has
Example request
{ "job_skills": ["Python", "Docker", "Kubernetes"], "candidate_skills": ["Python", "Docker"] }
Response 200
{ "missing_skills": ["Kubernetes"] }
Courses
POST /recommend-courses Recommend courses by skills ▼

Given a candidate's skills, return the top matching courses ranked by skill overlap and rating. Score = (matching skills / total skills) × 70% + (rating / 5) × 30%.

Request body
FieldTypeRequiredDescription
candidate_skillsstring[]requiredSkills to find courses for
job_skillsstring[]optionalUnused in this endpoint — pass []
Query parameter
ParamTypeDescription
top_nintegerMax courses to return. Default: 6
Example request
{ "job_skills": [], "candidate_skills": ["python", "fastapi", "postgresql"] }
Response 200
{ "skills": ["python", "fastapi"], "total": 2, "courses": [ { "id": 1, "title": "FastAPI Complete Course", "description": "...", "skills": ["python", "fastapi"], "rating": 4.8, "level": "Intermediate", "duration_hours": 12, "instructor": "", "match_score": 0.9760 } ] }
Error responses
422 — Provide at least one skill 502 — Course recommender error
CV Enhancer
POST /enhance Enhance resume to match a job ▼

Upload a resume file (.txt or .pdf) and provide a job description. Returns an LLM-enhanced version of the resume tailored to the role. Content-Type must be multipart/form-data.

Form fields
FieldTypeRequiredDescription
resumefilerequiredResume file (.txt or .pdf)
job_descriptionstringrequiredTarget job description as plain text
Example (curl)
curl -X POST http://localhost:8000/enhance \ -F "resume=@my_cv.pdf" \ -F "job_description=We are looking for a Senior Python Engineer..."
Response 200
{ "enhanced_resume": "Jane Doe\n\nEnhanced for the role of...", "model_used": "mistralai/mistral-small-2603", "reasoning_strategy": "Self-Ask" }
Error responses
422 — Resume or job description is empty 502 — LLM call failed
